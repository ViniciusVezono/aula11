1. Sistema de Administração de Medicamentos

O sistema visa auxiliar a gestão de medicamentos em um ambiente hospitalar, facilitando o trabalho de médicos e enfermeiros e garantindo a segurança dos pacientes.

1.1. Funcionalidades Principais:

Cadastro de usuários
Permite o cadastro de médicos e enfermeiros, com seus respectivos dados de identificação e acesso ao sistema.
Cadastro de pacientes
Realiza o registro de pacientes, com um leito específico.
Prescrição de medicamentos
Permite aos médicos prescrever medicamentos para os pacientes, definindo a dosagem, e a data e hora de administração.
Gerenciamento de receitas
Permite aos enfermeiros visualizarem as receitas pendentes e registrarem a administração dos medicamentos.

1.2. Cadastro de médicos:

Implemente um formulário de cadastro para médicos, coletando as seguintes informações:

Nome, Especialidade, CRM, usuario, senha

1.3. Exercícios

  1. Crie a página que recebe os dados do médico e salva na tabela medicos.
  2. Crie a página de login de um médico. Guarde em $_SESSION a informação de que é um médico.

1.4. Cadastro de pacientes:

O sistema deve permitir o cadastro de um paciente por médicos e enfermeiros logados inserindo os seguintes dados do paciente:

Nome, Leito

1.5. Exercícios

  1. Crie a página que cadastra um paciente na tabela pacientes.

1.6. Cadastro de Receitas Médicas:

Implemente usando formulário uma página para que os médicos possam cadastrar novas receitas, contendo os seguintes campos:

Nome do paciente, Nome do medicamento, Data da administração, Hora da administração, Dose.

1.7. Exercício

  1. Crie a página que recebe o nome do paciente e verifica se o paciente já foi cadastrado, se foi permita o cadastro de uma receita médica. Caso contrário, o médico deve cadastrar o paciente.

1.8. Cadastro de Enfermeiros:

Implemente um formulário para o cadastro de novos enfermeiros, coletando as seguintes informações:

Nome, COREN, usuario, senha

1.9. Exercício

  1. Crie a página que recebe os dados do enfermeiro e salva na tabela enfermeiros.
  2. Crie a página de login de um enfermeiro. Guarde em $_SESSION a informação de que é um enfermeiro.

1.10. O sistema deve permitir ao enfermeiro ver as receitas pendentes:

(Data, Hora, Leito)

1.11. Exercício

  1. Crie a página que mostra todas as receitas pendentes de administração.

1.12. O sistema deve permitir ao enfermeiro registrar uma administração:

Nome do paciente, Nome do medicamento, Data da administração, Hora da administração, Dose, Data e Hora do registro.

1.13. Exercício

  1. A página que mostra todas as receitas deve permitir ao usuário clicar em uma e registrar a administração da receita.

1.14. O sistema deve conter ao menos as seguintes tabelas:

medicos, pacientes, enfermeiros, receitas, administracoes.

1.15. Exercício

  1. Exporte o banco de dados SQL.
